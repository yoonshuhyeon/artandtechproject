# 요:앵이팅 (Yo:Aengting) - 실시간 과팅 도우미

과팅 현장에서 실시간으로 자리 배치, 대화 주제 추천, 긴급 도움 요청 등을 도와주는 웹 애플리케이션 프로토타입입니다.

## 🚀 주요 기능

### 1. 방 관리 및 초대
- **방 생성**: 대표가 장소, 시간, 예약자명을 입력하여 방을 생성 (룸 코드 직접 지정 가능).
- **초대 시스템**: 룸 코드 공유, 초대 링크 복사, 실시간 QR 코드 생성을 통한 간편 입장.
- **세션 유지**: `localStorage`를 활용하여 새로고침 시에도 입장 상태 유지.

### 2. 고도화된 자리 배치 (사랑의 짝대기) 알고리즘
모든 참여자가 지목을 완료하면 다음 우선순위에 따라 매칭이 수행됩니다:
1. **서로 지목 (Mutual Match)**: 남녀가 서로를 선택한 경우 최우선 매칭.
2. **여자 우선 (Female Choice)**: 남은 인원 중 여자가 선택한 남자가 비어있으면 매칭.
3. **남자 우선 (Concentration Handling)**: 여러 여자가 한 남자를 지목했을 경우, 그 남자가 지목한 여자와 우선 매칭 (남자의 선택권 존중).
4. **랜덤 매칭**: 지목이 엇갈리거나 선택하지 않은 경우 무작위 매칭.

### 3. 실시간 도우미 기능
- **아이스브레이킹**: 분위기에 맞는 질문 카드(가벼운/깊은/재미있는) 제공.
- **SOS 긴급 요청**: 화장실 가기, 지인 호출 등 곤란한 상황에서 사용할 수 있는 몰래 도움 요청 기능.
- **실시간 상태**: 현재 방에 접속한 남녀 인원 수 실시간 확인.

## 🛠 기술 스택
- **Backend**: Python (FastAPI), Dataclasses
- **Frontend**: HTML5, Vanilla CSS, JavaScript (Single Page Application 구조)
- **Deployment**: Vercel 지원 (`vercel.json`, `requirements.txt`)

## 📁 프로젝트 구조
- `api/main.py`: FastAPI 서버 로직 및 매칭 알고리즘.
- `static/`: 프론트엔드 리소스 (index.html, script.js, style.css).
- `assets/`: 서비스 UI용 이미지 및 아이콘.
- `requirements.txt`: 프로젝트 의존성.

## 🏃 실행 방법
```bash
# 서버 실행
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```
접속 주소: `http://localhost:8000`

## 📝 참고 사항
- 본 프로젝트는 프로토타입으로, 현재 데이터는 서버 메모리(`ROOMS` dict)에 저장됩니다. 서버 재시작 시 데이터가 초기화됩니다.
- 실운영 시 Redis 또는 Database 도입이 필요합니다.
